dojo.contentHandlers (version 1.6)

Object » dojo.contentHandlers
dojo.require("dojo._base.xhr");
Defined in dojo/_base/xhr.js

A map of availble XHR transport handle types. Name matches the handleAs attribute passed to XHR calls. Each contentHandler is called, passing the xhr object for manipulation. The return value from the contentHandler will be passed to the load or handle functions defined in the original xhr call.

Examples:

Example 1

Creating a custom content-handler: 
	dojo.contentHandlers.makeCaps = function(xhr){
		return xhr.responseText.toUpperCase();
	}
	// and later:
	dojo.xhrGet({
		url:"foo.txt",
		handleAs:"makeCaps",
		load: function(data){ /* data is a toUpper version of foo.txt */ }
	});

Method Summary

  • dojo.contentHandlers.javascript(xhr) A contentHandler which evaluates the response data, expecting it to be valid JavaScript
  • dojo.contentHandlers.json(xhr) A contentHandler which returns a JavaScript object created from the response data
  • dojo.contentHandlers.json-comment-filtered(xhr) A contentHandler which expects comment-filtered JSON.
  • dojo.contentHandlers.json-comment-optional(xhr) A contentHandler which checks the presence of comment-filtered JSON and alternates between the json and json-comment-filtered contentHandlers.
  • dojo.contentHandlers.text(xhr) A contentHandler which simply returns the plaintext response data
  • dojo.contentHandlers.xml(xhr) returns DOMDocument A contentHandler returning an XML Document parsed from the response data

dojo.contentHandlers.json-comment-filtered
Defined by dojo.contentHandlers

A contentHandler which expects comment-filtered JSON. the json-comment-filtered option was implemented to prevent "JavaScript Hijacking", but it is less secure than standard JSON. Use standard JSON instead. JSON prefixing can be used to subvert hijacking. Will throw a notice suggesting to use application/json mimetype, as json-commenting can introduce security issues. To decrease the chances of hijacking, use the standard json contentHandler, and prefix your "JSON" with: {}&& use djConfig.useCommentedJson = true to turn off the notice
