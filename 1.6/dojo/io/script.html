dojo.io.script (version 1.6)

Object » dojo.io.script

Property Summary

  • dojo.io.script._counter
  • dojo.io.script._deadScripts

Method Summary

  • dojo.io.script._addDeadScript(ioArgs) sets up an entry in the deadScripts array.
  • dojo.io.script._canAttach(ioArgs) provides an override of dojo.io.script._canAttach to check for the existence of a the args.frameDoc property. If it is there, and it is a string, not a document, then create the iframe with an ID of frameDoc, and use that for the calls. If frameDoc is a document, then dojox.io.scriptFrame should not get involved.
  • dojo.io.script._deferredCancel(dfd) canceller function for dojo._ioSetArgs call.
  • dojo.io.script._deferredError(error, dfd) errHandler function for dojo._ioSetArgs call.
  • dojo.io.script._deferredOk(dfd) okHandler function for dojo._ioSetArgs call.
  • dojo.io.script._ioCheck(dfd) inflight check function to see if IO finished.
  • dojo.io.script._makeScriptDeferred(args) returns dojo.Deferred sets up a Deferred object for an IO request.
  • dojo.io.script._resHandle(dfd) inflight function to handle a completed response.
  • dojo.io.script._validCheck(dfd) inflight check function to see if dfd is still valid.
  • dojo.io.script.attach(id, url, frameDocument) creates a new <script> tag pointing to the specified URL and adds it to the document.
  • dojo.io.script.get(args) sends a get request using a dynamically created script tag.
  • dojo.io.script.remove(id, frameDocument) removes the script element with the given id, from the given frameDocument. If no frameDocument is passed, the current document is used.

If this is used, then the content property is ignored. This is mostly useful for HTTP methods that have a body to their requests, like PUT or POST. This property can be used instead of postData and putData for dojo.rawXhrPost and dojo.rawXhrPut respectively.
