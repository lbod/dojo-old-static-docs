dojo.store.Cache (version 1.6)

Object » dojo.store.Cache
dojo.require("dojo.store.Cache");
Defined in dojo/store/Cache.js

Usage:

var foo = new dojo.store.Cache(/* This */ masterStore, /* This */ cachingStore, /* dojo.store.__CacheArgs */ options);

The Cache store wrapper takes a master store and a caching store, caches data from the master into the caching store for faster lookup. Normally one would use a memory store for the caching store and a server store like JsonRest for the master store.

Examples:

Example 1

	var master = new dojo.store.Memory(data);
	var cacher = new dojo.store.Memory();
	var store = new dojo.store.Cache(master, cacher);

Method Summary

  • add(object, directives) Add the given object to the store.
  • constructor(masterStore, cachingStore, options) returns dojo.store.Cache is the authoritative store, all uncached requests or non-safe requests will be made against this store.
  • evict(id) Remove the object with the given id from the underlying caching store.
  • get(id, directives) Get the object with the specific id.
  • put(object, directives) Put the object into the store (similar to an HTTP PUT).
  • query(query, directives) Query the underlying master store and cache any results.
  • remove(id, directives) Remove the object with the specific id.
