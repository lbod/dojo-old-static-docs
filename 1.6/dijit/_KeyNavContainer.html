dijit._KeyNavContainer (version 1.6)

Object » dijit._Container » dijit._KeyNavContainer
dojo.require("dijit._KeyNavContainer");
Defined in dijit/_KeyNavContainer.js

Usage:

var foo = new dijit._KeyNavContainer();

To use this mixin, call connectKeyNavHandlers() in postCreate() and call startupKeyNavChildren() in startup(). It provides normalized keyboard and focusing code for Container widgets.

Property Summary

  • _keyNavCodes
  • containerNode
  • focusedChild The currently focused child widget, or null if there isn't one
  • isContainer Indicates that this widget acts as a "parent" to the descendant widgets. When the parent is started it will call startup() on the child widgets. See also isLayoutContainer.
  • tabIndex Tab index of the container; same as HTML tabIndex attribute. Note then when user tabs into the container, focus is immediately moved to the first item in the container.

Method Summary

  • _getFirstFocusableChild() returns dijit._Widget Returns first child that can be focused
  • _getLastFocusableChild() returns dijit._Widget Returns last child that can be focused
  • _getNextFocusableChild(child, dir) returns dijit._Widget Returns the next or previous focusable child, compared to "child"
  • _getSiblingOfChild(child, dir) returns dijit._Widget Get the next or previous widget sibling of child
  • _startupChild(widget) Setup for each child widget
  • addChild(widget, insertIndex) Add a child to our _Container
  • buildRendering()
  • connectKeyNavHandlers(prevKeyCodes, nextKeyCodes) Call in postCreate() to attach the keyboard handlers to the container. preKeyCodes: dojo.keys[] Key codes for navigating to the previous child.
  • constructor() returns dijit._KeyNavContainer A reference to the constructor function for this object.
  • destroyDescendants(preserveDom) Destroys all the widgets inside this.containerNode, but not this widget itself
  • focus() Default focus() implementation: focus the first child.
  • focusChild(widget, last) Focus widget.
  • focusFirstChild() Focus the first focusable child in the container.
  • focusLastChild() Focus the last focusable child in the container.
  • focusNext() Focus the next widget
  • focusPrev() Focus the last focusable node in the previous widget (ex: go to the ComboButton icon section rather than button section)
  • getIndexOfChild(child) returns int Gets the index of the child in this container or -1 if not found
  • hasChildren() returns Boolean Returns true if widget has children, i.e. if this.containerNode contains something.
  • removeChild(widget) Removes the passed widget instance from this widget but does not destroy it. You can also pass in an integer indicating the index within the container to remove
  • startup() Called after all the widgets have been instantiated and their dom nodes have been inserted somewhere under dojo.doc.body. Widgets should override this method to do any initialization dependent on other widgets existing, and then call this superclass method to finish things off. startup() in subclasses shouldn't do anything size related because the size of the widget hasn't been set yet.
  • startupKeyNavChildren() Call in startup() to set child tabindexes to -1

Event Summary

  • _onBlur(evt)
  • _onChildBlur(widget) Called when focus leaves a child widget to go to a sibling widget.
  • _onContainerFocus(evt) Handler for when the container gets focus
  • _onContainerKeypress(evt) When a key is pressed, if it's an arrow key etc. then it's handled here.
