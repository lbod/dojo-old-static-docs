Allow end-users to track a DOM node into the web page

Property Summary

  • _firstX
  • _firstY
  • _isDragging
  • _selectStart
  • autoScroll
  • d
  • date
  • dragDistance The user clicks on the handle, but the drag action will really begin if he tracks the main node to more than 3 pixels.
  • events
  • handle The node on which the user clicks to drag the main node.
  • size
  • skip A flag to control a drag action if a form element has been focused. If true, the drag action is not executed.

Method Summary

  • constructor(params, node) returns dojox.mdnd.Moveable Configure parameters and listen to mousedown events from handle node.
  • destroy() Delecte associated events
  • initOffsetDrag(e) Initialize the gap between main node coordinates and the clicked point. Call the onDragStart method.
  • isFormElement(e) returns Boolean identify the type of target node associated with a DOM event.

Event Summary

  • onDrag(node, coords, size, mousePosition) Stub function. Notes : border box model for size value, margin box model for coordinates
  • onDragEnd(node) Stub function Notes : Coordinates don't contain margins
  • onDragStart(node, coords, size) Stub function. Notes : border box model
  • onFirstMove(e) Occurs when the user moves the mouse after clicking on the handle. Determinate when the drag action will have to begin (see dragDistance).
  • onMouseDown(e) Occurs when the user clicks on the handle node. Skip the drag action if a specific node is targeted. Listens to mouseup and mousemove events on to the HTML document.
  • onMouseUp(e) Occurs when the user releases the mouse Calls the onDragEnd method.
  • onMove(e) Occurs when the user moves the mouse. Calls the onDrag method.
