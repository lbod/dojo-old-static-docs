Method Summary

  • dojox.mobile.app.connectFlick(target, context, method) Listens for a flick event on a DOM node. If the mouse/touch moves more than 15 pixels in any given direction it is a flick. The synthetic event fired specifies the direction as <ul> <li><b>'ltr'</b> Left To Right</li> <li><b>'rtl'</b> Right To Left</li> <li><b>'ttb'</b> Top To Bottom</li> <li><b>'btt'</b> Bottom To Top</li> </ul>
  • dojox.mobile.app.getActiveSceneController() Gets the controller for the active scene.
  • dojox.mobile.app.getStageController() Gets the stage controller.
  • dojox.mobile.app.init(node) Initializes the mobile app. Creates the
  • dojox.mobile.app.loadResources(resources, callback)
  • dojox.mobile.app.loadResourcesForScene(sceneName, callback)
  • dojox.mobile.app.resolveAssistant(sceneName) Given the name of a scene, returns the path to it's assistant file. For example, for a scene named 'main', the file returned is 'app/assistants/main-assistant.js' This function can be overridden if it is desired to have a different name to file mapping.
  • dojox.mobile.app.resolveTemplate(sceneName) Given the name of a scene, returns the path to it's template file. For example, for a scene named 'main', the file returned is 'app/views/main/main-scene.html' This function can be overridden if it is desired to have a different name to file mapping.
