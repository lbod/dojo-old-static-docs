Usage:

var foo = new dojox.data.FileStore(/* Object */ args);

A simple store that provides a datastore interface to a filesystem. It takes a few parameters for initialization:

Examples:

Example 1

options="expand,dirsOnly,showHiddenFiles"

Property Summary

  • _attributes Internal variable of attributes all file items should have.
  • _identifier Default attribute to use to represent the item's identifier. Path should always be unique in the store instance.
  • _storeRef Internal variable used to denote an item came from this store instance.
  • failOk Flag to pass on to xhr functions to check if we are OK to fail the call silently
  • label string Default attribute to use to represent the item as a user-readable string. Public, so users can change it.
  • options Array of options to always send when doing requests. Back end service controls this, like 'dirsOnly', 'showHiddenFiles', 'expandChildren', etc.
  • pathAsQueryParam
  • pathSeparator The path separator to use when chaining requests for children Can be overriden by the server on initial load
  • url string The URL to the file path service.
  • urlPreventCache Flag to dennote if preventCache should be passed to xhrGet. Function to switch between REST style URL lookups and passing the path to specific items as a query param: 'path'.
