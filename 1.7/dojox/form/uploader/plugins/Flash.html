dojox.form.uploader.plugins.Flash (version 1.7)

Object » dojox.form.uploader.plugins.Flash
dojo.require("dojox.form.uploader.plugins.Flash");
Defined in dojox/form/uploader/plugins/Flash.js

Usage:

var foo = new dojox.form.uploader.plugins.Flash();

Inherits all properties from dojox.form.Uploader and dojox.form.uploader.plugins.HTML5. All properties and methods listed here are specific to the Flash plugin only.

Property Summary

  • _cons
  • _createInput
  • _fileMap
  • _files
  • _subs
  • deferredUploading (1 - X) (Flash only) throttles the upload to a certain amount of files at a time. By default, Flash uploads file one at a time to the server, but in parallel. Firefox will try to queue all files at once, leading to problems. Set this to the amount to upload in parallel at a time. Generally, 1 should work fine, but you can experiment with queuing more than one at a time. This is of course ignored if selectMultipleFiles equals false.
  • devMode Re-implemented. devMode increases the logging, adding style tracing from the SWF.
  • fieldname
  • force Use "flash" to always use Flash (and hopefully force the user to download the plugin if they don't have it).
  • getFileList
  • inputNode
  • isDebug If true, outputs traces from the SWF to console. What exactly gets passed is very relative, and depends upon what traces have been left in the DEFT SWF.
  • reset
  • serverTimeout (milliseconds) The amount of time given to the uploaded file to wait for a server response. After this amount of time, the onComplete is fired but with a 'server timeout' error in the returned item.
  • skipServerCheck If true, will not verify that the server was sent the correct format. This can be safely set to true. The purpose of the server side check is mainly to show the dev if they've implemented the different returns correctly.
  • submit
  • swfPath Path to SWF. Can be overwritten or provided in djConfig.
  • upload
  • uploadType

  • postMixInProperties()

Event Summary

  • _onFlashBlur(fileArray)
  • onFileChange(fileArray) Stub - Flash-specific event. Fires on each selection of files and only provides the files selected on that event - not all files selected, as with HTML5
  • onFileProgress(fileArray) Stub - Flash-specific event. Fires on progress of upload and only provides a file-specific event
  • onLoad(uploader) Stub - SWF has been downloaded 100%.
  • onReady(uploader) Stub - Fired when dojox.embed.Flash has created the Flash object, but it has not necessarilly finished downloading, and is ready to be communicated with.
