dijit/form/_DateTimeTextBox.__Constraints (version 1.8)

Specifies both the rules on valid/invalid values (first/last date/time allowed), and also formatting options for how the date/time is displayed.

See the dijit/form/_DateTimeTextBox.__Constraints reference documentation for more information.

Examples

Example 1

To restrict to dates within 2004, displayed in a long format like "December 25, 2005":

    {min:'2004-01-01',max:'2004-12-31', formatLength:'long'}

Property Summary

  • am override strings for am in times
  • datePattern override pattern with this string
  • formatLength choice of long, short, medium or full (plus any custom additions).
  • fullYear (format only) use 4 digit years whenever 2 digit years are called for
  • locale override the locale used to determine formatting rules
  • max Maximum signed value.
  • min Minimum signed value.
  • pm override strings for pm in times
  • selector choice of 'time','date' (default: date and time)
  • strict (parse only) strict parsing, off by default
  • timePattern override pattern with this string
