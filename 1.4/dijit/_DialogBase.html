dijit._DialogBase (version 1.4)

Object » dijit._Templated » dijit._DialogBase
dojo.require("dijit.Dialog");
Defined in dijit/Dialog.js

Usage:

var foo = new dijit._DialogBase();

Pops up a modal dialog window, blocking access to the screen and also graying out the screen Dialog is extended from ContentPane so it supports all the same parameters (href, etc.)

Examples:

Example 1

	<div dojoType="dijit.Dialog" href="test.html"></div>

Example 2

	var foo = new dijit.Dialog({ title: "test dialog", content: "test content" };
	dojo.body().appendChild(foo.domNode);
	foo.startup();
dijit.form._FormMixin, dijit._DialogMixin

  • domNode
  • draggable Toggles the moveable aspect of the Dialog. If true, Dialog can be dragged by it's title. If false it will remain centered in the viewport.
  • open True if Dialog is currently displayed on screen.
  • refocus A Toggle to modify the default focus behavior of a Dialog, which is to re-focus the element which had focus before being opened. False will disable refocusing. Default: true
  • templatePath Path to template (HTML file) for this widget relative to dojo.baseUrl. Deprecated: use templateString with dojo.cache() instead.
  • templateString A string that represents the widget template. Pre-empts the templatePath. In builds that have their strings "interned", the templatePath is converted to an inline templateString, thereby preventing a synchronous network call. Use in conjunction with dojo.cache() to load from a file.
  • underlayAttrs
  • widgetsInTemplate Should we parse the template to find widgets that might be declared in markup inside it? False by default.

Method Summary

  • _attachTemplateNodes(rootNode, getAttrFunc) Iterate through the template and attach functions and nodes accordingly.
  • _endDrag(e) Called after dragging the Dialog. Saves the position of the dialog in the viewport.
  • _fillContent(source) Relocate source contents to templated container node. this.containerNode must be able to receive children, or exceptions will be thrown.
  • _getFocusItems(dialogNode) Find focusable Items each time a dialog is opened, setting _firstFocusItem and _lastFocusItem
  • _getValueAttr() Returns Object representing form values.
  • _position() Position modal dialog in the viewport. If no relative offset in the viewport has been determined (by dragging, for instance), center the node. Otherwise, use the Dialog's stored relative offset, and position the node to top: left: values based on the viewport.
  • _setValueAttr(obj) Fill in form values from according to an Object (in the format returned by attr('value'))
  • _setup() Stuff we need to do before showing the Dialog for the first time (but we defer it until right beforehand, for performance reasons).
  • _size() If necessary, shrink dialog contents so dialog fits in viewport
  • _stringRepl(tmpl) Does substitution of ${foo} type properties in template string
  • _widgetChange(widget) Connected to a widget's onChange function - update our valid state, if needed.
  • buildRendering() Construct the UI for this widget from a template, setting this.domNode.
  • connectChildren() Connects to the onChange function of all children to track valid state changes. You can call this function directly, ex. in the event that you programmatically add a widget to the form *after* the form has been initialized.
  • constructor() returns dijit._DialogBase A reference to the constructor function for this object.
  • destroy()
  • destroyRendering()
  • execute(formContents) Callback when the user hits the submit button. Override this method to handle Dialog execution.
  • getValues()
  • hide() Hide the dialog
  • isValid() Returns true if all of the widgets are valid
  • layout() Position the Dialog and the underlay
  • postCreate()
  • postMixInProperties()
  • reset()
  • setValues(val)
  • show() Display the dialog
  • startup()
  • uninitialize()
  • validate() returns if the form is valid - same as isValid - but provides a few additional (ui-specific) features. 1 - it will highlight any sub-widgets that are not valid 2 - it will call focus() on the first invalid sub-widget

Event Summary

  • _onCloseEnter() Called when user hovers over close icon
  • _onCloseLeave() Called when user stops hovering over close icon
  • _onKey(evt) returns just let it go Handles the keyboard events for accessibility reasons
  • _onSubmit() Callback when user hits submit button type: protected
  • onCancel() Called when user has pressed the Dialog's cancel button, to notify container.
  • onExecute() Called when user has pressed the dialog's OK button, to notify container.
  • onLoad() Called when data has been loaded from an href. Unlike most other callbacks, this function can be connected to (via dojo.connect) but should *not* be overriden.
  • onValidStateChange(isValid) Stub function to connect to if you want to do something (like disable/enable a submit button) when the valid state changes on the form as a whole.

_getValueAttr
Defined by dijit.form._FormMixin

Returns name/value hash for each form element. If there are multiple elements w/the same name, value is an array, unless they are radio buttons in which case value is a scalar since only one can be checked at a time. If the name is a dot separated list (like a.b.c.d), creates a nested structure. Only works on widget form elements.

Example 1
{ name: "John Smith", interests: ["sports", "movies"] }
