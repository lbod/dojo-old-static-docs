dijit._Container (version 1.5)

Object » dijit._Container
dojo.require("dijit._Container");
Defined in dijit/_Container.js

Usage:

var foo = new dijit._Container();

Use this mixin for widgets that needs to know about and keep track of their widget children. Suitable for widgets like BorderContainer and TabContainer which contain (only) a set of child widgets. It's not suitable for widgets like ContentPane which contains mixed HTML (plain DOM nodes in addition to widgets), and where contained widgets are not necessarily directly below this.containerNode. In that case calls like addChild(node, position) wouldn't make sense.

Property Summary

  • containerNode
  • isContainer Indicates that this widget acts as a "parent" to the descendant widgets. When the parent is started it will call startup() on the child widgets. See also isLayoutContainer.

Method Summary

  • _getSiblingOfChild(child, dir) returns dijit._Widget Get the next or previous widget sibling of child
  • addChild(widget, insertIndex) Makes the given widget a child of this widget.
  • buildRendering()
  • constructor() returns dijit._Container A reference to the constructor function for this object.
  • destroyDescendants(preserveDom) Destroys all the widgets inside this.containerNode, but not this widget itself
  • getIndexOfChild(child) returns int Gets the index of the child in this container or -1 if not found
  • hasChildren() returns Boolean Returns true if widget has children, i.e. if this.containerNode contains something.
  • removeChild(widget) Removes the passed widget instance from this widget but does not destroy it. You can also pass in an integer indicating the index within the container to remove
  • startup() Called after all the widgets have been instantiated and their dom nodes have been inserted somewhere under dojo.doc.body. Widgets should override this method to do any initialization dependent on other widgets existing, and then call this superclass method to finish things off. startup() in subclasses shouldn't do anything size related because the size of the widget hasn't been set yet.
